# ControlIventario
# Sistema de Control de Inventario

Este proyecto consiste en el desarrollo de un sistema de control de inventario diseñado para ser intuitivo y automatizar procesos críticos como el registro de productos y el control de movimientos. El sistema busca reducir errores humanos y proporcionar información confiable para la toma de decisiones en un negocio.

## Objetivos del Proyecto
**Automatización**: Registrar productos y actualizar cantidades disponibles de forma eficiente.
**Seguridad**: Implementar un sistema de roles (Administrador y Empleado) para evitar alteraciones indebidas.
**Análisis**: Generar estadísticas y reportes sobre productos con mayor rotación.
**Movilidad**: Permitir la consulta del inventario en tiempo real desde dispositivos móviles.

## Requerimientos Principales
De acuerdo con la especificación de requerimientos, el software cumple con:
**Gestión de Usuarios**: Registro e inicio de sesión mediante correo y contraseña.
**Control de Inventario**: Registro detallado por nombre, categoría, lote y fecha.
**Validación de Datos**: No se permiten cantidades negativas y se garantiza la integridad de los datos.
**Auditoría**: Registro de cada movimiento detallando fecha, usuario y tipo de acción].

## Especificaciones Técnicas
**Paradigma**: Programación Orientada a Objetos (POO) para facilitar el mantenimiento[cite: 129].
**Seguridad**: Control de acceso donde solo el administrador puede modificar cantidades manualmente o asignar roles.
**Usabilidad**: Interfaz intuitiva para usuarios sin conocimientos técnicos avanzados[cite: 119].
